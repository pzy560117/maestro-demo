# LLM 驱动的手机端 UI 自动化定位系统 — 界面设计说明

## 1. 设计概述

- **设计身份**：UI 设计师
- **设计风格**：采用 **Bento 布局 + Glassmorphism 进阶暗黑模式**，局部配合动态文字排版与柔和弥散光渐变，以模块化卡片呈现复杂数据，强调层次与空间感。
- **色彩方案**：
  - 主色（Primary）`#4C6FFF`（Electric Indigo，Bento 卡片及关键按钮背景）
  - 辅助色（Secondary）`#1B1F2B`（玻璃拟态暗色背景，提供沉浸式基底）
  - 强调色（Accent）`#5FE1B7`（Neon Mint，用于高亮指标、状态标签、关键 CTA 辅助色）
  - 渐变参考：`linear-gradient(135deg, rgba(76,111,255,0.85), rgba(95,225,183,0.65))`
- **排版系统**：使用 **Inter / Source Han Sans** 系列
  - 标题尺寸：`32px`（Bento 主区块标题），次级标题 `24px`
  - 正文尺寸：`16px`（主要内容），辅助说明 `14px`
- **视觉元素**：
  - 图标风格：FontAwesome Pro Duotone / 线性图标，配合 Bootstrap Icons，用渐变描边增强玻璃质感。
  - 插图风格：半透明渐变矢量插图（可来自 unDraw / Lottie 动效），贴合自动化、AI、棋盘式流程主题。
  - 照片风格：如需实景，使用 Unsplash 深色调科技场景，覆盖玻璃模糊层，保持统一光感。
- **设计原则**：层次分明（Bento 栅格）、视觉引导（渐变光束 + 对比色 CTA）、一致性（统一阴影、圆角、玻璃拟态模糊值）。
- **前端框架约束**：所有交互/组件按照 Bootstrap 5 栅格与变量设计，图标统一使用 FontAwesome，保留可扩展到其他开源组件库的空间。

## 2. 主要页面信息架构

### 2.1 仪表盘（Dashboard）

- **布局**：
  - 顶部 12 列全宽资讯条（任务统计、LLM Token 使用、设备状态概览）
  - 中部 Bento 卡片分布：`6+6` 列（遍历覆盖率、定位成功率）、`4+4+4` 列（三个 KPI 卡片）
  - 下部使用 8/4 分栏呈现「最新任务活动」与「告警时间线」
  - 玻璃拟态卡片 24px 圆角，背景 `rgba(27,31,43,0.65)` + `backdrop-filter: blur(18px)`
- **关键模块**：
  - KPI Bento | 数字+趋势折线+微粒散射动效
  - 任务列表 | Bootstrap `table-responsive` + 自适应列
  - 告警时间线 | Vertical Stepper + Accent 色斑点
- **交互**：
  - KPI 卡片悬停：提升阴影（`0 24px 40px rgba(76,111,255,0.25)`），数字轻微缩放 1.03
  - 告警列表 hover：背景浅亮 (`rgba(95,225,183,0.12)`)，右侧出现 CTA icon

### 2.2 遍历任务列表 / 详情

- **列表页**：
  - 顶部过滤器区域（Bootstrap Accordion + Chips）
  - 数据表格使用 `table` + `sticky` 表头 + 状态徽章
  - 行内提供「查看详情」「重新运行」「导出」按钮，使用 FontAwesome 图标
- **详情页**：
  - 左侧固定信息卡（任务基础、设备）、右侧 Tab（操作时间线、定位验证、LLM 日志）
  - 操作时间线嵌套 Stepper + 代码块风格 JSON 预览
  - 回放区使用 Glassmorphism 视频容器（支持 3D 光泽覆盖）

### 2.3 界面版本库 & 定位详情

- **界面库**：Bento Gallery，卡片内展示缩略图 + 版本信息 + 最新定位准确率
- **界面详情**：
  - 顶部展示大图（支持 2x Retina），侧边 Info Panel（定位列表、置信度曲线）
  - Tab：`定位策略 | 历史验证 | 差异对比`

### 2.4 告警中心

- **布局**：
  - 左侧过滤器 + 状态统计
  - 右侧列表为 Card + Tag 组合，强调 severity
  - 告警详情弹窗：Glass 模态，内含处理历史线

## 3. 组件系统（Bootstrap 兼容）

| 组件 | 说明 | 状态 |
| --- | --- | --- |
| 导航栏（Navbar） | 顶部固定，透明背景滚动渐变，高度 72px，右侧包含用户菜单与暗黑切换 | 默认、滚动收缩、激活 tab |
| 侧边导航（Sidebar） | 玻璃拟态面板 + Icon 标签 + Tooltip，支持折叠 | 展开、折叠、悬停高亮 |
| KPI 卡片 | Bento 模块，内含图标、数值、趋势微图 | 默认、悬停、点击钻取 |
| 数据表格 | Bootstrap Table + Badge + Tag + 分页条 | 行悬停、排序、选中、禁用 |
| 筛选 Chips | Pills 样式，支持多选/单选 | 默认、hover、selected |
| 表单控件 | Bootstrap Form + Floating Label，玻璃背景 | 默认、聚焦、错误、禁用 |
| CTA 按钮 | Primary / Secondary / Ghost | 默认、hover、active、loading、disabled |
| Toast & Snackbar | 底部右侧浮层，玻璃背景 + Glow | 成功、警告、错误 |
| 模态框 | 中央玻璃卡片 + 背景模糊 | 默认、滚动、全屏 |
| 标签页（Tabs） | Pill 样式 + 下划线渐变 | 默认、hover、active |

### 状态风格定义

- **按钮**：
  - 默认：背景主色 + 72% 透明白色边缘高光
  - Hover：背景加深 8%，投影扩散
  - Active：背景压暗 12%，内阴影表示按压
  - Loading：按钮文本替换为 Spinner + 文本
- **表单**：
  - 聚焦：外发光 `0 0 0 4px rgba(95,225,183,0.25)`，标签文字上浮
  - 错误：边框 `#FF6B6B`，下方展示帮助文本 + FontAwesome `fa-triangle-exclamation`
- **Toast**：成功 `#5FE1B7` 渐变、错误 `#FF6B6B`、警告 `#F4A259`

### 微交互动效建议

- KPI 数字使用 200ms 数值跃迁动画（ease-out）
- 告警时间线节点在首次呈现时从下向上淡入，伴随轻微光晕扩散
- 数据表格排序图标 120° 旋转，时长 160ms
- 导航切换时使用 280ms 滑动条过渡

## 4. 交互设计与反馈

- **关键交互状态**：
  - 卡片：默认玻璃背景，hover 提升高度 + 轻微摇曳光斑；点击进入详情
  - 图标按钮：交互态背景 `rgba(76,111,255,0.12)`, active 背景主色 40% 透明
  - Tabs：hover 下划线渐变延展，active 下划线厚度 4px
- **表单交互**：
  - 必填项标记 `*`，实时验证（Blur 触发），错误信息在 200ms 内淡入
  - 成功状态：边框变为 Accent 色，浮动文字变绿，出现打勾图标
- **反馈机制**：
  - 成功提交任务：显示成功 toast + 页面顶部 Notification 插入卡片
  - 执行失败：表单顶部危险提示条 + 涉及字段标红 + 建议操作链接
- **错误 / 成功状态**：
  - 错误卡片使用 `#FF6B6B` 渐变 + `fa-circle-exclamation`
  - 成功使用 `#5FE1B7` Glow + `fa-circle-check`

## 5. 响应式设计策略

- **断点定义（基于 Bootstrap）**：
  - `≥1920px`（超大屏）：Bento 区域增加列间距至 32px，展示更多统计卡片
  - `≥1440px`：标准桌面布局（12 列，卡片 6/3/3 组合）
  - `≥1280px`：侧边栏收缩为图标模式，Bento 改为 6/6 + 12 分布
  - `≥1024px`（横屏平板）：顶部信息条改为横向滚动 Chips，表格进入紧凑模式
  - `≥768px`：Bento 卡片堆叠为 6 -> 12，侧栏变顶部菜单
  - `428px / 375px`：移动端采用卡片纵向流，表格转化为卡片列表（每条展示关键字段 + CTA 二级菜单）
- **响应式要点**：
  - 使用 Bootstrap `col-xxl/col-xl/col-lg/col-md/col-sm/col` 组合
  - 过滤器在移动端折叠为底部弹出 Sheet
  - KPI 卡片在移动端采用横向可滚动容器，提供 `snap` 效果

## 6. 可访问性规范（WCAG 2.1 AA）

- **色彩对比**：主色与文字对比度 ≥ 4.5:1，玻璃背景叠加深色遮罩保证文本可读
- **键盘可导航**：
  - 所有交互控件（按钮、tabs、chips、表格操作）支持 `Tab` 顺序与 `Space/Enter` 激活
  - 焦点可见样式：`outline: 2px solid #5FE1B7; outline-offset: 4px`
- **替代文本**：
  - 所有图标按钮提供 `aria-label`
  - 截图缩略图标注界面签名、版本等描述
- **屏幕阅读器兼容**：
  - KPI 卡片使用 `role="status" aria-live="polite"`
  - 表格使用 `<caption>` 与 `<th scope>`
- **动效控制**：提供“减少动态效果”开关，遵循 `prefers-reduced-motion`

## 7. 图片与图标规范

- **图标**：FontAwesome / Bootstrap Icons 优先，保证矢量 SVG，可通过 `fa-duotone` 增强层次
- **插图 / 图片**：
  - 来自 Unsplash / unDraw，统一暗调+玻璃遮罩
  - 使用 `WebP`、`AVIF` 格式，最大宽度 1200px；提供 1x / 2x 资源
  - 避免占位符，必要时使用抽象 AI 插图
- **性能优化**：
  - 利用图像懒加载（Intersection Observer）
  - 精简 Lottie 动效帧数并延迟加载

## 8. 设计资产交付

- 矢量：Figma 组件库（Bootstrap 变量 + Glassmorphism 样式）
- 字体：Inter / Source Han Sans (OTF)，提供 WebFont 与备用字体（`font-family: 'Inter', 'PingFang SC', 'Microsoft Yahei', sans-serif;`）
- 色板：主色、辅色、强调色、背景（`#0E111A`）、文字（主 `#F6F7FB`、次 `#9BA8C7`）
- 阴影：`0 24px 48px rgba(12,15,24,0.45)` 基础阴影，Hover 增强版 `rgba(76,111,255,0.25)`

## 9. 后续工作建议

1. 基于本设计说明在 Figma 建立完整 Design System（含组件、样式、动效原型）
2. 与前端协作定义 Bootstrap 主题变量（例如 `--bs-primary`、`--bs-body-bg`）
3. 准备光栅资源的导出策略（`@1x/@2x`、WebP）并交付资源清单
4. 设计易用的无障碍模式切换（高对比度、减少动效）
